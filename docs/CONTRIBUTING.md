# Contributing to Community Medicine Manual

Thank you for your interest in contributing to the Community Medicine Manual! This document provides guidelines and information for contributors.

## Ways to Contribute

### 1. Report Issues
- Found a bug or error in the content?
- Have suggestions for improvements?
- Missing important information?

Please [open an issue](https://github.com/yourusername/community-medicine-manual/issues) with detailed information.

### 2. Suggest Content Improvements
- Propose new topics or sections
- Suggest restructuring of existing content
- Recommend additional references or resources

### 3. Submit Pull Requests
- Fix typos, grammatical errors, or formatting issues
- Add new content or update existing information
- Improve documentation or scripts

## Getting Started

### Prerequisites
- Basic knowledge of Markdown formatting
- Understanding of Community Medicine concepts
- Git and GitHub account

### Development Setup
1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/yourusername/community-medicine-manual.git
   cd community-medicine-manual
   ```
3. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Content Guidelines

### Writing Style
- Use clear, concise language appropriate for medical students
- Maintain professional and academic tone
- Use consistent terminology throughout
- Include relevant medical abbreviations with full forms on first use

### Structure and Formatting
- Follow the existing chapter structure and formatting
- Use proper Markdown syntax
- Include headings, subheadings, and bullet points for readability
- Add tables where appropriate for comparisons or data

### Content Accuracy
- Ensure all medical information is accurate and up-to-date
- Cite reliable sources and references
- Base content on established medical literature
- Include evidence-based recommendations where applicable

### References and Citations
- Include references for key statements and data
- Use consistent citation format
- Update bibliography when adding new references

## Pull Request Process

1. **Create a Branch**: Create a descriptive branch name for your changes
2. **Make Changes**: Implement your improvements or additions
3. **Test Locally**: Build the PDF locally to ensure formatting is correct
4. **Commit Changes**: Use clear, descriptive commit messages
5. **Push to GitHub**: Push your branch to your fork
6. **Open Pull Request**: Create a pull request with a detailed description

### Pull Request Template
When opening a pull request, please include:
- **Description**: What changes were made and why
- **Type of Change**: Bug fix, content addition, formatting, etc.
- **Testing**: How the changes were tested
- **Screenshots**: If applicable (PDF output, etc.)

## Code of Conduct

### Professional Standards
- Maintain respectful and professional communication
- Be constructive in feedback and reviews
- Respect diverse perspectives and experiences
- Focus on improving the educational resource

### Academic Integrity
- Do not plagiarize content
- Properly attribute sources and references
- Ensure original work or properly licensed content
- Maintain academic honesty in all contributions

## Review Process

### Content Review
- Changes will be reviewed by maintainers
- Medical accuracy will be verified
- Formatting and consistency will be checked
- Feedback will be provided for improvements

### Approval and Merging
- Pull requests require approval from maintainers
- Major changes may require additional review
- Once approved, changes will be merged into the main branch

## Recognition

Contributors will be recognized in:
- Repository contributors list
- Acknowledgments section in the manual
- Release notes for significant contributions

## Questions?

If you have questions about contributing:
- Check existing issues and documentation
- Open a discussion issue
- Contact the maintainers

Thank you for helping improve the Community Medicine Manual!
